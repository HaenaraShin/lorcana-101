# 디즈니 로카나 101 — 1차 슬라이드 설계 보고서

> **대상 산출물**: `index.html` + `styles.css` + `app.js` + `cards.js`
> **이번 차수 범위**: Chapter 0 ~ 7 (인트로 ~ 메인 페이즈 3대 핵심 액션)
> **총 슬라이드 수**: 25장
> **기술 스택**: Reveal.js 5.1.0 (CDN) · Vanilla JS · Ravensburger CDN 카드 이미지
> **작성일**: 2026-05-21

이 문서는 1차로 만든 25장 슬라이드 각각에 대해 **어떤 이유로 / 어떤 관점에서 / 어떤 출처를 근거로** 구성했는지를 정리한 설계 보고서입니다. 2차 작업 시 챕터별 가지치기·확장에 활용합니다.

---

## 0. 설계 원칙 (전체 슬라이드를 관통하는 결정)

| 원칙 | 결정 | 근거 |
|---|---|---|
| **순서** | 스타터덱 룰북(Whispers)의 챕터 순서를 100% 따른다 | 라벤스부르거가 자체 검증한 학습 곡선. 재발명 금지 |
| **시작점** | 룰이 아니라 **세계관·승리 조건**부터 | 룰북 1p가 *Winning*과 *About the Game*으로 시작 |
| **카드 분류 어휘** | 4타입 + Song 서브타입 + Location | 룰북·s1 가이드 둘 다 4타입 체계 |
| **잉크 색 표현** | 6가지 색의 *성격*까지 가르침 | s1 가이드 "Choose Your Inks Wisely" |
| **챌린지 학습** | 룰북 예시(Headless Horseman vs Rapunzel)를 **인터랙티브 시뮬레이터**로 변환 | 스타터덱 룰북 14~15p의 결투 예시 그대로 |
| **"카드가 룰보다 강하다"** | 1챕터 마지막에 단독 슬라이드로 강조 | 룰북 1p에 명시된 핵심 메타룰 |
| **메인 페이즈** | "절차"가 아닌 **"메뉴판"** 컨셉으로 시각화 | 룰북의 *any number of times, in any order* 철학 |
| **시각 일관성** | 다크 테마 + 황금(Lorcana 잉크 색조) 강조 | TCG 분위기 + 야간 강습 환경 가독성 |
| **이미지 소스** | Ravensburger 공식 CDN 직접 사용 | JSON에 URL 임베드 + 별도 호스팅 불필요 |
| **인터랙티브 핵심** | 카드 해부도 호버 / 챌린지 시뮬레이터 두 곳에 집중 | 웹 PPT의 최대 강점 = 정적 슬라이드보다 우월한 부분 |

## 챕터별 설계 의도

### 📂 Chapter 0 — 로카나 세계로 (3슬라이드)

스타터덱 룰북은 표지 한 페이지에 *"As an Illumineer in the wondrous realm of Lorcana... Endless quests await!"* 라는 단 3문장으로 세계관을 압축한다. 이 미니멀리즘을 따라 챕터 0도 **딱 3장으로 정서적 진입 장벽을 무너뜨린다**.

#### 슬라이드 1: 표지 — "로카나 입문 101"

- **목적**: 시작 신호. 어떤 강습인지 정체성 부여.
- **구성**: 아이브로(eyebrow) 카피("Disney Lorcana TCG · 초심자 강습") + 큰 타이틀 + 부제 + 키보드 힌트
- **출처**: 스타터덱 표지 페이지의 톤 차용
- **디자인 결정**: 카드 이미지를 의도적으로 배제 → 첫인상은 "텍스트와 분위기"만으로

#### 슬라이드 2: 너는 Illumineer다

- **목적**: 플레이어 정체성과 핵심 어휘 4종(Illumineer/Glimmer/Story Star/Lore) 학습
- **구성**: 좌측 내러티브 + 우측 어휘 박스 (정의 카드)
- **출처**: s1 Player's Guide 2~3p의 세계관 설명을 압축
- **디자인 결정**: 좌측은 *읽는* 영역, 우측은 *참조하는* 영역으로 역할 분리

#### 슬라이드 3: 우리의 목표

- **목적**: 승리 조건 "20"이라는 숫자를 **시각적 임팩트**로 각인
- **구성**: 거대한 "20" 숫자 + 보조 텍스트 + 푸터의 본질 한 줄
- **출처**: 룰북 1p의 *Winning* 섹션을 한 슬라이드로 압축
- **디자인 결정**: 12em 크기의 황금색 숫자. *"공격해서 깎는 게임"이 아닌 "승점 누적 레이스"* 라는 메시지를 푸터에 박음 — 이게 가장 자주 오해되는 부분

### 📂 Chapter 1 — 승리 조건과 게임의 본질 (2슬라이드)

룰북이 *Winning → About the Game → What Kind of Game Is This?* 순서로 푸는 흐름을 압축. 핵심은 **승리의 두 가지 경로**와 **"카드가 룰보다 강하다"** 메타룰.

#### 슬라이드 4: 승리하는 두 가지 방법

- **목적**: 주승리 조건(20 lore)과 보조 승리(덱 소진) 모두 인지
- **구성**: 2열 카드. 주승리는 황금 테두리로 강조, 보조는 평범한 카드
- **출처**: 룰북 19p *Ending the Game*
- **디자인 결정**: "1순위 vs 2순위"의 시각적 위계 — 주승리만 황금 테두리. 초보가 첫 게임에서 신경 쓸 것은 1번뿐임을 암시

#### 슬라이드 5: 가장 중요한 룰 하나

- **목적**: *"카드 텍스트가 게임 규칙과 충돌하면 카드를 따른다"* 메타룰 각인
- **구성**: 영어 원문 인용 + 한국어 번역 + 푸터에 의미 해설
- **출처**: 룰북 1p *About the Game* 마지막 문장
- **디자인 결정**: **단독 슬라이드**로 분리한 이유 — 이 한 줄이 신규 카드 출시 시 룰북 미숙지 플레이어의 가장 큰 안전망이기 때문. 별도 강조 가치 충분

### 📂 Chapter 2 — 카드는 네 종류 + 노래 + 장소 (6슬라이드)

룰북 3~7p의 *The Cards* 섹션 구조를 따른다. 개요 1장 + 타입별 카드 5장. **실제 카드 이미지를 보여주는 첫 챕터** — 텍스트 설명 위주에서 시각 자료 위주로 전환되는 분기점.

#### 슬라이드 6: 카드는 4종류

- **목적**: 4타입의 존재 자체를 한 눈에 인지
- **구성**: 4열 그리드 + 아이콘 + 한 줄 설명 + 푸터에 Song 언급
- **출처**: 룰북 3~6p 구조
- **디자인 결정**: 4타입을 동등하게 4열 그리드로 배치. Song은 Action의 *서브타입*이므로 별개 컬럼이 아닌 푸터 노트로 배치 — 룰북의 분류 철학 그대로

#### 슬라이드 7: Character — 게임의 주역

- **목적**: 캐릭터가 *퀘스트와 챌린지를 모두 한다*는 핵심 + "잉크가 마를 때까지" 제약 인지
- **카드 예시**: **Maui - Hero to All** (set 1)
- **출처**: 룰북 3p + s1 가이드의 Maui 추천
- **디자인 결정**: Maui를 고른 이유 — Rush + 강한 stats를 모두 가진 캐릭터. 이후 키워드 챕터(2차)에 자연스럽게 연결되는 카드

#### 슬라이드 8: Item — 필드에 남는 장비

- **목적**: Item이 캐릭터와 달리 *즉시 사용 가능*함을 인지
- **카드 예시**: **Magic Golden Flower** (set 1)
- **출처**: 룰북 3p + 4p
- **디자인 결정**: Magic Golden Flower는 일러스트가 직관적이고 *"치유"* 효과로 Item의 보조적 성격을 잘 보여줌

#### 슬라이드 9: Action — 1회용 마법

- **목적**: Action의 *한 번 쓰고 묘지* 특성 + 강력한 효과
- **카드 예시**: **Dragon Fire** (set 1)
- **출처**: 룰북 4p + s1 가이드의 Top 12에서 Dragon Fire 추천
- **디자인 결정**: Dragon Fire는 *banish a character* 라는 가장 직관적 효과 → Action의 "강력함"을 한 카드로 전달

#### 슬라이드 10: Song — 잉크 없이도 부를 수 있다

- **목적**: Song의 *대체 발동 방식*(캐릭터 Exert)을 학습
- **카드 예시**: **Friends on the Other Side** (set 1)
- **출처**: 룰북 4p + s1 가이드 추천 카드
- **디자인 결정**:
  - 카드 옆에 *공식 비유* (`5코스트 노래 = 5코스트↑ 캐릭터 Exert`)를 시각화
  - Friends on the Other Side가 *대량 드로우 + Song*이라 1차 학습용으로 가장 임팩트 큼

#### 슬라이드 11: Location — 가만히 있어도 로어를 만든다

- **목적**: Location 타입의 *수동 로어 획득* + 챌린지 가능하지만 반격 안 함
- **카드 예시**: **The Great Illuminary - Abandoned Laboratory** (set 10)
- **출처**: 룰북 16~17p *Locations* 전용 페이지
- **디자인 결정**: 일루미너리는 세계관 챕터(0)에서 이미 언급한 *Great Illuminary*의 글리머. **세계관과 메커니즘이 연결**되는 의도적 카드 선택

### 📂 Chapter 3 — 카드 한 장 해부도 (2슬라이드)

**1차 작업의 시각적 하이라이트**. 룰북 8~9p의 *Parts of a Card* 페이지를 인터랙티브 호버로 변환.

#### 슬라이드 12: 카드 한 장 해부도 (인터랙티브)

- **목적**: 카드의 9개 영역을 *호버로 학습*
- **카드**: **The Headless Horseman - Terror of Sleepy Hollow** (set 10)
- **출처**: 룰북 8~9p의 9개 라벨링 영역
- **인터랙션**: 카드 위 9개 영역을 hover/click 시 우측 정보 박스에 설명 표시
- **디자인 결정**:
  - 시뮬레이터에서도 쓰는 Horseman 카드를 *재사용* → 학습 일관성
  - 호버 시 황금색 빛으로 영역 표시 → 정적 PDF로는 절대 불가능한 인터랙션
  - 9개 영역 좌표는 Lorcana 표준 카드 레이아웃에 맞춰 정밀 보정 (1차 작업 중 1회 재조정 완료)

#### 슬라이드 13: 잉크화 가능 vs 불가능

- **목적**: 잉크웰 아이콘의 의미와 덱빌딩 함의
- **구성**: 좌(가능)/우(불가능) 대칭 비교
- **출처**: 룰북 8p의 *Inkwell Icon* 라벨 + s1 가이드 4p의 *"비잉크 카드 16장 이하"* 권장
- **디자인 결정**: 좌측은 황금 테두리(긍정), 우측은 파선(부정)으로 시각적 구분. 푸터에 16장 가이드라인까지 포함 → 덱빌딩 챕터로 자연스러운 다리

### 📂 Chapter 4 — 잉크 시스템 (3슬라이드)

룰북 12~13p의 *Your Inkwell* 섹션 + *Ready vs Exerted* 다이어그램 + s1 가이드의 잉크 색 가이드를 통합.

#### 슬라이드 14: 잉크 — 모든 자원의 시작

- **목적**: 다른 TCG와의 **차별점** 인지 (땅/에너지 없음, 손의 어떤 카드든 잉크 가능)
- **구성**: 좌(차별점) / 우(룰)
- **출처**: 룰북 12p
- **디자인 결정**: MTG/포켓몬 경험자를 위한 mental model bridge — 좌측에 명시적으로 *"MTG의 땅, 포켓몬의 에너지"* 비교

#### 슬라이드 15: Ready와 Exerted

- **목적**: 카드 상태 토글 메커니즘 시각화
- **구성**: Ready 카드 ↔ Exerted 카드 (90도 회전) + 화살표
- **출처**: 룰북 13p의 Ready/Exerted 다이어그램
- **디자인 결정**: 실제 카드 회전을 CSS transform으로 표현 → 룰북의 정적 일러스트를 동적 데모로 변환

#### 슬라이드 16: 6가지 잉크 색의 성격

- **목적**: 각 잉크 색의 *전략적 정체성* 인지
- **구성**: 3×2 그리드. 각 색은 해당 컬러 토큰으로 시각화
- **출처**: s1 가이드 4~5p *Choose Your Inks Wisely*
- **디자인 결정**:
  - 룰북에 거의 없는 정보지만, 덱빌딩의 핵심이기에 1차에 포함
  - 각 색을 한 줄 설명으로 압축 (예: Ruby = "공격적·러시·제거")
  - 푸터에 *"덱은 최대 2색까지"* 룰 명시 → 다음 챕터(덱빌딩)로 다리

### 📂 Chapter 5 — 게임 준비 + 턴 구조 (2슬라이드)

룰북 10~11p의 *Setting Up* + *Playing the Game, Your Turn*.

#### 슬라이드 17: 게임 준비 5단계

- **목적**: 첫 게임에서 *그냥 따라하면 되는* 5단계 절차
- **구성**: 순서 있는 리스트 + 콜아웃("첫 게임은 멀리건 스킵")
- **출처**: 룰북 10p
- **디자인 결정**: 룰북의 *"첫 게임에선 멀리건 건너뛰세요"* 친절함을 콜아웃 박스로 보존 → 강습회 첫 실전에 직결

#### 슬라이드 18: 턴 구조 — Ready, Set, Draw

- **목적**: Beginning Phase 3단계 + Main Phase의 메뉴판 컨셉 학습
- **구성**: 위(Beginning Phase 3단계 카드) + 아래(Main Phase 액션 메뉴)
- **출처**: 룰북 11p + 마지막 페이지 *TURN ORDER AT A GLANCE*
- **디자인 결정**:
  - Beginning은 *번호가 있는 단계*로, Main은 *번호 없는 칩들*로 표현 → "순서 강제 vs 자유 선택"의 의미적 구분
  - 푸터에 *"절차가 아니라 메뉴판"* 핵심 메시지

### 📂 Chapter 6 — 3대 핵심 액션 (6슬라이드)

**1차 작업의 가장 큰 챕터**. 룰북 14~15p의 *Questing/Challenging/Damage* 전체를 6슬라이드로 확장.

#### 슬라이드 19: 메인 페이즈의 3대 핵심

- **목적**: Quest/Challenge/Sing 세 액션의 *역할 구분* (점수/견제/효율)
- **구성**: 3열 카드 + 각 액션의 한 줄 정의
- **출처**: 룰북 14~15p + Songs 페이지 통합
- **디자인 결정**: 각 액션의 "역할 라벨"(점수/견제/효율)을 명시 → s1 가이드의 *카드 역할 분류* 사고법을 액션에도 적용

#### 슬라이드 20: Quest — 승리의 정도(正道)

- **목적**: 퀘스트 방법 + 제약 이해
- **카드 예시**: **Mickey Mouse - Brave Little Tailor** (set 1)
- **출처**: 룰북 14p
- **디자인 결정**: Mickey - Brave Little Tailor는 *Lore 4*의 고로어 캐릭터 → 퀘스트의 위력을 한 카드로 보여줌

#### 슬라이드 21: Challenge — 전투의 작동 방식 (인터랙티브 시뮬레이터)

- **목적**: 챌린지의 *동시 데미지 처리*를 실제로 보여주기
- **카드 예시**: **The Headless Horseman vs Rapunzel - Creative Captor**
- **출처**: **스타터덱 룰북 14~15p의 실제 예시 그대로**
- **인터랙션**: 3단계 진행 버튼
  1. 챌린지 선언
  2. 동시 데미지 입힘 (둘 다 빨간 카운터 표시)
  3. 결과 판정 (Horseman 추방, Rapunzel 4 데미지로 생존)
- **디자인 결정**:
  - **이 슬라이드가 1차 작업의 최대 임팩트 포인트**. 룰북에서 가장 헷갈리는 부분(*동시 데미지*)을 정확히 그 룰북의 예시 카드를 가지고 *움직이는 데모*로 만듦
  - 결과까지 룰북과 100% 일치 (수치는 카드 DB에서 자동 추출)
  - Banish 애니메이션 (회색조 + 회전 + 투명도)
  - 데미지 카운터 오버레이 (실제 게임 토큰을 시뮬레이트)

#### 슬라이드 22: Challenge — 알아야 할 룰

- **목적**: 시뮬레이터에서 보여주지 못한 *예외/제약 룰*들을 보강
- **구성**: 2열 그리드 8항목 (✅/❌/⚔️/💀/🩸/⏳ 등 이모지 라벨)
- **출처**: 룰북 14~15p의 챌린지 룰 + 키워드 챕터 일부
- **디자인 결정**: 시뮬레이터로 *왜*를 보여줬으니, 이 슬라이드는 *조건*을 정리 — *시각적 이해 → 텍스트 정리*의 학습 시퀀스

#### 슬라이드 23: Sing — 잉크를 아끼는 마법

- **목적**: 노래의 *경제학* — 잉크 절약 효과
- **카드 예시**: **Friends on the Other Side** (set 1)
- **출처**: 룰북 13p + s1 가이드의 Friends 카드 설명
- **디자인 결정**: Slide 10에서 이미 보여준 카드를 *다시 사용* — 첫 등장은 "타입 소개", 이번에는 "전략적 활용". 같은 카드의 다층 학습

#### 슬라이드 24: Sing Together — 합창

## 기술적 결정 사항

### Reveal.js 설정
- `hash: false` — 새로고침 시 항상 첫 슬라이드로 (강습 진행 중 사고 방지)
- `slideNumber: 'c/t'` — 현재/전체 표시
- `width: 1400, height: 900` — 16:9에 가까운 자유 비율 (강습장 모니터 호환성)
- `center: true` — 컨텐츠가 적은 슬라이드도 가운데 정렬

### 카드 데이터
- `cards.js`는 Python 추출 스크립트로 자동 생성 (재현 가능)
- 카드 이미지는 Ravensburger 공식 CDN 직접 참조 — 별도 호스팅 불필요
- 1차에 사용된 카드: 13장 (Character 4 + Item 2 + Action 1 + Song 3 + Location 1 + Challenge 예시 2)

### 인터랙션 구현
- **카드 해부도**: 9개 절대 위치 hotspot div, 호버/클릭 시 active 클래스 + 사이드 정보 패널 업데이트
- **챌린지 시뮬레이터**: 상태 머신 4단계 (idle → 선언 → 데미지 → 판정). 카드 DB의 실제 stats 사용

### CSS 시스템
- 6개 잉크 색 토큰 + 다크 배경
- 카드 컴포넌트는 종횡비 5:7 강제 (`width: 320px; height: 446px`)
- 모바일 대응: 1024px 이하에서 single-column fallback
